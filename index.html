include.js

A pure JavaScript micro library designed to make it easy to include other JavaScript files.

  • Easy to use
  • Lightweight
  • Cross browser

Including include.js

You can either download the latest version of include.js from here and use it as follows:

<src="include.js"></script>

Or you can link to the latest version as follows:

<script src="http://felisphasma.github.io/include.js/include.js"></script>

Ussage

Including single files:

include("foo.js", function(){
	// When foo.js has loaded
	console.log("bar");
});

Including multiple files

include(["foo0.js", "foo1.js", "foo2.js", "foo3.js"], function(){
	// When foo0.js has loaded
	console.log("bar0");
}, function(){
	// When foo1.js has loaded
	console.log("bar1");
},function(){
	// When foo2.js has loaded
	console.log("bar2");
},function(){
	// When foo3.js has loaded
	console.log("bar3");
});

include.js handles functions modularly. the diret code: callbacks[i % callbacks.length]. This means that Callbacks can alternate like this:

include(["foo0.js", "foo1.js", "foo2.js", "foo3.js"], function(){
	// Called for foo0.js and foo2.js
	console.log("bar");
}, function(){
	// Called for foo1.js and foo3.js
	console.log("bar");
});

Or the same callback for all

include(["foo0.js", "foo1.js", "foo2.js", "foo3.js"], function(){
	// Called each time one of the files loads
	console.log("bar");
});

Include Method

include.js can include files using two different methods. Ajax and dynamic. Though they mostly serve the same purpose, they do work differently. Ajax makes a call too the url and uses eval() to run the code. The ajax method cannot include files across domains. Dynamic creates a new script tag and appends it to the head tag (or another selector specified in the configuration).

The default method is dynamic but that can be changed in configuration. Each method can be accessed and used the same way like so:

include.ajax(file, callback);
include.ajax([files], callback1, callback2, ...);

include.dynamic(file, callback);
include.dynamic([files], callback1, callback2, ...);

Configuration

Configurations:

Name Datatype Value
defaultMethod String ajax tells include.js to include the file through ajax. dynamic tells include.js to include the file by appending a script tag.
ajaxAsync bool Tells include.js to use asynchronous or nonasynchronous include requests.
includeHolder string A query to select the item that scripts are appended to (dynamic method). Selectors are passed through SparkleQuery (a micro JavaScript selector library) and supports three selectors: tag, #id, and .class. When multiple items have been selected, include.js uses the first one.

There are two ways to configure include.js. The first configures include.js at launch (code must go before you include include.js):

window.includejs = {
	defaultMethod: string,
	ajaxAsync: bool,
	includeHolder: string
};

Or you can configure include.js after launch:

include.config({
	defaultMethod: string,
	ajaxAsync: bool,
	includeHolder: string
});

Ajax warning

Including files via ajax works very well for non-cross doemain requests. But do be warned, however, that eval() is used and can be dangerous. Exerpt from https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/eval:

eval() is a dangerous function, which executes the code it's passed with the privileges of the caller. If you run eval() with a string that could be affected by a malicious party, you may end up running malicious code on the user's machine with the permissions of your webpage / extension. More importantly, third party code can see the scope in which eval() was invoked, which can lead to possible attacks in ways to which the similar Function is not susceptible.
